# Further Reading

The sources this skill is built on, plus related collections. Fetch these when a task calls for more depth than the bundled references provide — e.g. hunting for a niche principle set (service design, algorithmic systems, physical/built environment) not covered here.

## Primary sources bundled in this skill

- **[Principles of User Interface Design](http://bokardo.com/principles-of-user-interface-design/)** — Joshua Porter. The basis for `references/ui-principles.md`. Full explanations of each of the nineteen principles.
- **[Principles of Product Design](http://bokardo.com/principles-of-product-design/)** — Joshua Porter. The basis for `references/product-principles.md`. Full explanations of each of the eighteen principles.

## Broader collections (not bundled)

- **[Design Principles FTW](http://www.designprinciplesftw.com/)** — self-described as "the biggest collection of design principles on the internet." Useful when you need a specific niche set — service design, algorithm design, brand, writing, etc.
- **[principles.adactio.com](https://principles.adactio.com/)** — Jeremy Keith's curated list of personal and organizational design principles. Good for seeing how different disciplines frame their own values (W3C, Tim Berners-Lee, Dieter Rams, Bruce Tognazzini, Sandi Wassmer, Dave Winer, Lou Downe, and many more).
- **[Learn Design Principles](http://learndesignprinciples.com/)** — Applies design principles across domains — architecture, product, graphic, UX. Useful when thinking about principles that transcend software.

## Specialized

- **[Built Environment Innovation Design — Designer Roles](https://github.com/BEICOOP/BEICPBLChallenge/blob/master/Phase3/Stakeholders_Roles/Designer.md)** — Catalogue of professional design disciplines associated with built-environment ventures and their roles. Relevant when a task crosses into architecture, urban planning, physical product, or hardware — contexts where "designer" is a much broader term than in pure software.

## Specific principle sets worth knowing by name

When working on a specialized surface, it's worth searching directly for the canonical set for that domain. A non-exhaustive list:

- **Nielsen's 10 Usability Heuristics** — still the most cited UI heuristics set; overlaps heavily with Porter and Tognazzini.
- **Gestalt principles** — proximity, similarity, closure, continuity, figure/ground. Foundational for visual grouping.
- **Shneiderman's Eight Golden Rules** — another classical interaction-design set, strong on error handling and user control.
- **WCAG** — the accessibility standard. Not "principles" in the aesthetic sense, but a hard constraint for any real product.
- **Material Design / Apple Human Interface Guidelines** — platform-specific principles that also encode general wisdom.
- **Inclusive Design Principles (inclusivedesignprinciples.org)** — the Microsoft / Paciello Group version, complementary to the Heydon Pickering set in `timeless-principles.md`.

## How to search further

When the user's task hits a domain not covered here:

1. Start with principles.adactio.com and search for the domain keyword.
2. If nothing, designprinciplesftw.com as a second pass.
3. If still nothing, look for the canonical figure in the field (e.g. Edward Tufte for data viz, Don Norman for cognitive affordances, Jakob Nielsen for usability) and pull their principle set directly.

The goal isn't encyclopedic coverage — it's having enough vocabulary that a critique is grounded in established craft rather than AI-generated platitudes.
